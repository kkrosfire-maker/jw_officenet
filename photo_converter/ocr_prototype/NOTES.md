# OCR 그리드 프로토타입 — 질문과 진행 상황

**질문**: 3단계(그리드 OCR) 방향을 정하기 전에, 실제 사진(다양한 variation)에서
Tesseract가 어떤 결과를 내는지 확인하고, 빨간 테두리 영역을 (a) 줄 단위 raw
텍스트로 다룰지, (b) 컬럼 위치 기반으로 표를 파싱할지, (c) 다른 방식이 필요한지
판단한다.

**실행**: `python ocr_prototype/tui.py` (photo_converter 폴더에서)

## 확인된 것
- `find_red_rects`가 annotate.py가 그리는 순수 빨강(BGR 0,0,255) 테두리를 색상
  마스크로 정확히 감지함 (샘플 사진에서 2개 테두리 정확히 검출).
- 기본 설정(scale=1.0)에서는 원본 해상도가 커도(4134x3539) 글자가 촘촘해서
  인식률이 낮음. scale=2.0 업스케일 시 일부 개선되지만 여전히 오독 다수
  (예: "캡슐"→"캠슬", "0.2g"→"0.26", "393"→"93").
- 표 안에서 한 항목이 2줄로 줄바꿈된 셀(예: 약품명이 길어서 다음 줄로 넘어감)이
  있어서, row-clustering(y좌표 기준 줄 묶기)이 실제 표의 논리적 행과 어긋나는
  경우가 있음 — 컬럼 위치 기반 파싱만으로는 부족할 수 있음.

## 아직 정해지지 않은 것 (사용자가 직접 tui로 탐색 후 결정)
- scale/psm/contrast 조합 중 실사용 가능한 정확도가 나오는 조합이 있는지
- 컬럼 위치 비율(`setcol`)이 사진마다 얼마나 달라지는지 (다양한 variation 사진들로 테스트 필요)
- raw OCR + 수동 전체 교정이 나을지, 컬럼 자동 파싱 + 부분 교정이 나을지

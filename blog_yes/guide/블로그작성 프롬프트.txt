[[research 에이전트]]

agents/researcher.md 파일을 만들어줘.

이 에이전트는 사용자가 준 주제에 대해 웹 리서치를 수행하는 역할이야.

작동 방식:
- 주제를 받으면 웹 검색으로 충분한 정보를 수집
- 가능한 한 2026년 5월 현재 기준 최신 자료 위주로 (오래된 정보는 우선순위 낮게)
- 신뢰할 만한 소스 5개 이상 확인
- 수집한 정보를 정리해서 output/[주제]/research.md에 저장
- 정보는 글쓰기 에이전트가 그대로 활용할 수 있을 정도로 충실하게 정리할 것
  (단순 키워드 나열이 아니라 실제 내용이 담긴 정리)
- 소주제별로 섹션을 나누되, 구조는 주제 특성에 맞게 자유롭게 결정
- 마지막에 참고한 소스 URL 목록 첨부

지킬 것:
- 출처 없는 정보는 적지 않을 것
- 광고성 콘텐츠는 소스에서 제외
- 정보가 충돌하면 양쪽 의견을 다 기록
- 추측이나 창작 금지. 검색으로 확인한 것만 정리

산출물: output/[주제]/research.md 한 개
researcher 에이전트를 호출해서
"클로드 코드의 핵심 기능 5가지" 주제로 리서치 진행해줘.



[[글쓰기 에이전트]]

agents/writer.md 파일을 만들어줘.

이 에이전트는 리서치 결과와 가이드 파일들을 읽고 실제 블로그 글을 작성하는 역할이야.

입력:
- output/[주제]/research.md (리서치 결과)
- guides/style-guide.md (말투/문체 가이드 - 반드시 따를 것)
- guides/seo-guide.md (SEO 가이드 - 반드시 따를 것)

작동 방식:
- 위 세 파일을 모두 읽고 시작
- style-guide.md의 말투와 문체를 정확히 지킬 것
- seo-guide.md의 제목 작성법, 본문 구조, 키워드 배치를 따를 것
- research.md를 메인 소스로 활용. 보충 설명이나 자연스러운 흐름을 위해 너의 자체 지식을 함께 사용해도 됨
  (단, 사실 관계나 수치 등 검증이 필요한 정보는 research.md에 있는 것만 사용)
- 본문 중간중간 이미지가 들어갈 자리를 [IMAGE: 설명] 형태로 표시
  예: [IMAGE: 클로드 코드 활용 사례 5가지 비교 표]
  예: [IMAGE: 클로드 코드 워크플로우 단계별 다이어그램]

산출물: output/[주제]/draft.md
writer 에이전트를 호출해서
방금 만든 research.md를 기반으로 글을 작성해줘.






[[이미지 생성 에이전트]]

guides/image-guide.md 파일을 만들어줘.
이건 image-maker 에이전트가 읽을 가이드야.

내용:

[대표 이미지 규격]
- 가로 1080px, 세로 1080px (1:1 정사각형)
- 배경: 어두운 블루-퍼플 그라데이션 (#1a1a2e → #16213e)
- 메인 텍스트: 글 제목 (큰 글씨, 흰색)
- 보조 텍스트: 카테고리나 부제 (작은 글씨, 연한 회색)
- 모든 텍스트는 가로/세로 중앙 정렬
- 폰트: Pretendard 또는 시스템 기본 한글 폰트
- 우측 하단 또는 좌상단에 작은 액센트 (도형, 코드 심볼, 또는 단순한 아이콘)

[본문 삽입 이미지 종류 - 네 가지 중 글 내용에 맞게 선택]
1. 비교 표 (제품/방법을 나란히 비교할 때)
2. 단계별 다이어그램 (절차나 순서를 보여줄 때)
3. 핵심 포인트 카드 (3~5개 요점을 정리할 때)
4. 인용/강조 박스 (중요한 한 마디를 강조할 때)

[공통 규칙]
- 모두 HTML + CSS로 만들고, Python으로 Playwright를 실행해서 PNG로 캡처
- 배경 톤은 본문과 어울리게 (AI/테크 분위기)
- 텍스트는 최대한 짧게 (이미지는 보조 수단)
- 코드 블록 스타일을 디자인 요소로 활용 가능 (모노스페이스 폰트, 어두운 배경)
agents/image-maker.md 파일을 만들어줘.

이 에이전트는 글에 들어갈 이미지들을 HTML+CSS로 만들고
Python에서 Playwright를 실행해 PNG로 캡처하는 역할이야.

입력:
- output/[주제]/draft.md
- guides/image-guide.md

작동 방식:
1. draft.md를 읽고 [IMAGE: 설명] 마커들을 순서대로 모두 찾기
2. 글 제목과 분위기를 보고 대표 이미지 1장 만들기
   - output/[주제]/images/thumbnail.png 로 저장
3. 각 [IMAGE: ...] 마커마다 image-guide.md의 템플릿 중 적절한 걸 선택
4. 각 이미지를 HTML+CSS로 만들고 Python+Playwright로 PNG 캡처
   - 마커가 등장한 순서대로 body-1.png, body-2.png, ... 로 저장
5. 자체 검수 루프: 캡처한 PNG들을 view 도구로 직접 다시 읽어서 시각적으로 확인
   - 하단에 과도한 빈 여백이 있는지
   - 텍스트가 잘리거나 박스 밖으로 튀어나갔는지
   - 요소들이 비뚤어지거나 깨졌는지
   - 문제가 있으면 해당 이미지의 HTML/CSS를 수정하고 다시 캡처 → 다시 확인
   - 문제 없을 때까지 반복 (최대 3회)
6. draft.md의 [IMAGE: ...] 마커를 실제 이미지 경로로 직접 치환
   예: [IMAGE: 핵심 기능 비교 표] → ![핵심 기능 비교 표](./images/body-1.png)

[사용자 이미지 활용 - 옵션]
user-images/ 폴더에 사용자가 직접 찍거나 만든 이미지가 있으면:
- 각 이미지를 view 도구로 확인
- 글 내용 중 어디에 들어가면 자연스러울지 판단
- 해당 마커를 사용자 이미지로 대체
- alt 텍스트와 캡션은 이미지 분석 결과 기반으로 자동 생성 (SEO 효과)

산출물:
- output/[주제]/images/thumbnail.png, body-1.png, body-2.png, ...
- output/[주제]/draft.md (마커가 실제 이미지 경로로 치환된 상태)
image-maker 에이전트를 호출해서
방금 만든 draft.md의 이미지들을 만들어줘.



[[조립 에이전트]]

agents/assembler.md 파일을 만들어줘.

이 에이전트는 image-maker가 이미지 경로까지 다 박아둔 draft.md를 받아서
최종 미리보기 파일을 만드는 역할이야.

입력:
- output/[주제]/draft.md (image-maker가 이미지 경로까지 치환해둔 상태)
- output/[주제]/images/ 폴더

작동 방식:
1. draft.md를 읽기 (이미 마크다운 이미지 문법으로 경로가 박혀있음)
2. 두 가지 형식의 최종 파일 생성:
   - output/[주제]/final.md (마크다운 - 검토용)
   - output/[주제]/final.html (HTML - 시각 미리보기용)
3. final.html은 네이버 블로그 본문 영역과 비슷한 스타일링 적용
   - 본문 폭 700px 정도
   - 한글 가독성 좋은 폰트
   - 이미지는 본문 폭에 맞춰 자동 리사이즈
   - 제목, 소제목, 본문 위계가 명확하게

산출물:
- output/[주제]/final.md
- output/[주제]/final.html

마지막에 사용자에게 final.html을 브라우저로 열어보라고 안내할 것.
assembler 에이전트를 호출해서
글과 이미지를 합친 최종 파일을 만들어줘.



[[메인 에이전트]]

이 프로젝트의 CLAUDE.md를 만들어줘.

이 프로젝트는 블로그 글 작성 자동화 시스템이야.


메인 에이전트(너)는 직접 리서치나 글쓰기를 하지 않고,
서브 에이전트들을 호출해서 작업을 분배하는 오케스트레이터 역할이야.

CLAUDE.md에는 다음을 짧게 정리해줘:

1. 이 프로젝트의 목적 한 줄
2. 폴더 구조 안내 (agents/, guides/, output/)
3. 사용자가 주제를 던졌을 때 따라야 할 단계
   - Step 1: agents/researcher.md를 따라 리서치 → output/[주제]/research.md 생성
   - Step 2: agents/writer.md를 따라 글쓰기 → output/[주제]/draft.md 생성
   - Step 3: agents/image-maker.md를 따라 이미지 생성 → draft.md 마커 치환
   - Step 4: agents/assembler.md를 따라 통합 → final.html, final.md 생성
4. 각 단계 사이에 사용자에게 진행 상황을 짧게 알릴 것
5. 절대 메인이 직접 글을 쓰거나 리서치하지 말 것 (서브 에이전트에게 위임)

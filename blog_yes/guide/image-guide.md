# 블로그 이미지 제작 가이드

> 이 가이드는 연세예스내과 네이버 블로그에 삽입되는 모든 이미지의 제작 기준을 정의합니다.
> 이미지 제작 에이전트는 이 파일을 먼저 읽고 작업을 시작해야 합니다.

## 1. 대표 이미지 (썸네일) 규격

### 🚨 thumbnail 제작 핵심 원칙

> **참조 이미지(`1.png`)에서 제목 텍스트만 교체한다. 그 외 어떤 요소도 변경하지 않는다.**

thumbnail은 HTML/CSS로 새로 만드는 것이 아니라, 아래 방식으로 제작한다.

**제작 방법:**
1. 참조 이미지(`C:\Users\JW\Downloads\제목을 입력해주세요\1.png`)를 베이스로 사용
2. Python PIL로 기존 제목 텍스트 영역을 배경색(흰색)으로 덮기
3. 동일한 폰트·크기·색상·정렬로 새 제목 텍스트 쓰기
4. 1080×1080으로 리사이즈 (PIL `LANCZOS` 필터 사용)
5. 저장

**절대 변경하지 않아야 할 요소:**
- 클립보드 프레임 구조 (오렌지 배경 + 흰 종이 + 금속 클립)
- "함께 알아보는 의학정보" 카테고리 텍스트
- 흰 종이 좌하단 대각선 회색 그림자 요소
- 텍스트 우측 정렬 배치
- 연세예스내과 로고 위치·크기
- 전체 레이아웃 및 여백

**변경하는 것은 오직:**
- 제목 텍스트 내용 (예: "환절기 불청객 비염·감기" → "더워진 날씨 건강 주의사항 5가지")

| 항목 | 기준 |
|------|------|
| 크기 | 1080 × 1080 px (1:1 정사각형) |
| 포맷 | PNG |
| 배경 | 골든 오렌지-옐로우 그라디언트 (`#F5A623` → `#F0C040`, 상단→하단 또는 좌상→우하) |
| 프레임 | 클립보드 스타일 — 흰색 또는 연크림 둥근 사각형(border-radius: 24px) 내부 카드, 상단 중앙에 클립 심볼(직사각형 도형) |
| 카테고리 텍스트 | 카드 상단, 소형(28~32px), 흰색, "함께 알아보는 의학정보" 또는 글 카테고리 문구 |
| 메인 텍스트 | 카드 중앙, 대형(72~96px), 볼드, 진한 네이비(`#1A2340`) 또는 다크브라운(`#2C2010`), 2~3줄 |
| 보조 텍스트 | 카드 하단, 소형(24~28px), 중간 회색(`#666666`), 한 줄 부제 또는 생략 가능 |
| 브랜드 마크 | 우측 하단, 오렌지 원형 아이콘(`#F4803A`) + "연세예스내과" 텍스트(흰색, 20px) |
| 정렬 | 모든 텍스트 가로·세로 중앙 정렬 |
| 폰트 | Pretendard (없을 경우 Noto Sans KR 또는 시스템 기본 한글 폰트) |
| 액센트 요소 | 카드 상단 중앙에 클립 심볼(회색 둥근 직사각형) 1개 |

│   │                  │          │
│   │       ● 연세예스 │          │  ← 브랜드 마크 (우하단)

## 2. 본문 삽입 이미지 종류

글 내용에 맞게 아래 6가지 중 하나를 선택해 제작한다.

### 종류 1 — 비교 표
- **사용 시점**: 제품·방법·증상 등 두 가지 이상을 나란히 비교할 때
- **예시**: 열탈진 vs 열사병 증상 비교, 올바른 자세 vs 잘못된 자세
- **구성**: 항목명(헤더) + 좌우 또는 상하 비교 컬럼

### 종류 2 — 단계별 다이어그램
- **사용 시점**: 절차나 순서를 순차적으로 보여줄 때
- **예시**: 응급처치 순서, 올바른 손 씻기 6단계
- **구성**: 번호 + 아이콘/도형 + 짧은 설명, 화살표로 흐름 연결

### 종류 3 — 핵심 포인트 카드
- **사용 시점**: 3~5개 요점을 한눈에 정리할 때
- **예시**: 온열질환 예방 수칙 5가지, 수분 보충 체크리스트
- **구성**: 카드형 레이아웃, 번호 또는 아이콘 + 한 줄 텍스트

### 종류 4 — 인용·강조 박스
- **사용 시점**: 중요한 한 마디, 경고 문구, 핵심 수치를 강조할 때
- **예시**: "목이 마르면 이미 탈수입니다", 자외선 지수 9.6 강조
- **구성**: 큰 텍스트 중앙 배치, 배경색 또는 테두리로 강조

### 종류 5 — 실사 기반 AI 이미지
- **사용 시점**: 환자 상태, 진료 행위, 생활 장면 등 특정 행위를 사실적으로 묘사할 때
- **예시**: 더운 야외에서 어지러워하는 사람, 혈압 측정 장면, 물 마시는 장면
- **도구**: Gemini 또는 ChatGPT(DALL·E) — 무료 플랜 우선 사용
- **주의**: 특정 실존 인물, 병원 내부 사진, 의료진 모습은 사용 금지

### 종류 6 — 만화·일러스트 기반 AI 이미지
- **사용 시점**: 별도 요청이 있는 경우에만 사용
- **도구**: Gemini 또는 ChatGPT(DALL·E) — 무료 플랜 우선 사용
- **스타일**: 명시된 스타일이 없을 경우 부드럽고 친근한 플랫 일러스트 스타일

## 3. 공통 제작 규칙

### 🚨 본문 이미지 핵심 원칙

> **본문 이미지는 반드시 1:1 (1080×1080) 정사각형이어야 한다.**  
> **프레임 이미지(`클로드코드 교육용.png`)를 오버레이로 사용한다. HTML로 클립보드 프레임을 직접 만들지 않는다.**

**제작 방법 (3단계):**

1. **HTML 콘텐츠 작성** — `.clipboard` 래퍼는 HTML에 포함시키되, 실제 클립보드 CSS는 오버라이드로 제거한다.  
   HTML 파일은 `tmp_html/body-N.html`에 저장. 스타일은 **카드 내용 디자인만** 담당.

2. **Playwright 캡처** — 1080×1080 고정 뷰포트로 캡처 (`full_page=False`)

3. **PIL 합성** — 프레임 PNG(`클로드코드 교육용.png`)의 흰/회색 영역을 투명화한 뒤 캡처 이미지 위에 오버레이

**공유 스크립트:**
```powershell
python scripts/capture_bodies.py [주제]
```

**수동 합성 코드 (참고):**
```python
import numpy as np
from PIL import Image

FRAME_PNG = r"C:\Users\JW\Downloads\클로드코드 교육용.png"

# 프레임 투명화 (흰/회색 → 투명, 오렌지 프레임·클립은 유지)
frame = Image.open(FRAME_PNG).convert("RGBA")
arr = np.array(frame, dtype=np.int32)
is_light = (arr[:,:,0] > 225) & (arr[:,:,1] > 225) & (arr[:,:,2] > 225)
arr_u8 = arr.astype(np.uint8)
arr_u8[is_light, 3] = 0
transparent_frame = Image.fromarray(arr_u8, "RGBA")

# 캡처된 콘텐츠 이미지 위에 프레임 씌우기
base = Image.open(content_png_path).convert("RGBA")
base.alpha_composite(transparent_frame)
base.convert("RGB").save(output_path, "PNG")
```

**HTML 클래스명 규칙 (Convention B 단일 표준):**

> 반드시 아래 클래스명만 사용한다. capture_bodies.py가 이 이름을 기준으로 오버라이드를 적용한다.

| 역할 | 클래스명 |
|------|---------|
| 클립보드 외곽 래퍼 | `.clipboard-frame` |
| 클립(숨김 처리됨) | `.clip-wrapper` |
| 콘텐츠 카드 내부 | `.inner-card` |

**CSS 오버라이드 자동 주입 (`capture_bodies.py`가 처리):**

```css
html, body { width:1080px; height:1080px; overflow:hidden; background:white; padding:0; position:relative; }
.clipboard-frame {
  position: absolute;
  left: 85px; top: 88px;
  width: 907px; height: 903px;
  background: white; border-radius: 0; box-shadow: none;
  padding: 55px 50px 40px;
  overflow: hidden;
}
.clip-wrapper { display: none; }
.inner-card { background: transparent; border-radius: 0; padding: 0; }
```

**프레임 PNG 흰 영역 좌표 (픽셀 스캔 확인값):**
- 좌: x=85, 우: x=992, 상: y=88, 하: y=991
- 콘텐츠 영역: 907×903px (내부 여백 65px top, 55px horizontal, 45px bottom)

### 디자인 원칙
- **배경 톤**: 콘텐츠 내부 요소는 흰 배경 기준으로 배색 선택  
  - 병원·의료 정보: 밝고 깔끔한 흰색·연청색 계열
  - 경고·주의: 연한 주황·붉은 계열 강조색 포인트
  - 생활·일상 정보: 따뜻한 크림·베이지 계열
- **텍스트 원칙**: 이미지 내 텍스트는 최대한 짧게 작성
  - 한 줄 최대 20자 이내 권장
  - 핵심 단어·수치만 이미지에 표시, 상세 설명은 본문 텍스트로 처리
- **폰트**: Pretendard 우선, 없을 경우 Noto Sans KR 또는 시스템 기본 한글 폰트

### AI 이미지 생성 도구 우선순위
1. **Gemini** (Google AI Studio 무료 플랜) — 우선 시도
2. **ChatGPT DALL·E** (무료 플랜 한도 내) — 대안
3. 두 도구 모두 한도 초과 시 사용자에게 알리고 대기

## 4. `[IMAGE: 설명]` 플레이스홀더 해석 기준

`draft.md`에 삽입된 `[IMAGE: 설명]` 태그를 이미지로 변환할 때 아래 기준으로 종류를 판단한다.

| 플레이스홀더 설명 키워드 | 선택할 이미지 종류 |
|------------------------|-----------------|
| 비교, vs, 차이 | 종류 1 — 비교 표 |
| 단계, 순서, 방법, 절차 | 종류 2 — 단계별 다이어그램 |
| 수칙, 포인트, 정리, 체크 | 종류 3 — 핵심 포인트 카드 |
| 강조, 인용, 경고, 수치 | 종류 4 — 인용·강조 박스 |
| 장면, 모습, 상황, 환자, 진료 | 종류 5 — 실사 기반 AI 이미지 |
| 만화, 일러스트, 캐릭터 | 종류 6 — 만화·일러스트 AI 이미지 |

판단이 애매한 경우 **종류 3 (핵심 포인트 카드)** 를 기본값으로 선택한다.

## 5. 산출물 저장 경로

```
output/
└── [주제]/
    ├── research.md
    ├── draft.md
    ├── tmp_html/              ← 캡처용 HTML 중간 파일
    │   ├── body-1.html
    │   ├── body-2.html
    │   └── ...
    └── images/
        ├── thumbnail.png      ← 대표 이미지 (PIL로 1.png 텍스트 교체)
        ├── body-1.png
        ├── body-2.png
        └── body-N.png         ← 마지막은 4.png 복사 (클리닉 정보 카드)
```

이미지 파일명은 `body-1.png`, `body-2.png` 순서로 저장하며,
`draft.md`의 `[IMAGE: 설명]` 순서와 일치시킨다.

HTML 중간 파일은 반드시 `tmp_html/` 폴더에 저장한다.
공유 스크립트 `scripts/capture_bodies.py`가 이 폴더를 읽어 PNG를 생성한다.
